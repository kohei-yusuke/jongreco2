## 🐛 これまで（Before）の主な問題

| 領域 | 症状 |
|------|------|
| 精算ロジック | **3か所で別実装**され、画面ごとに点数が食い違っていた |
| ウマ | スコア履歴で**ウマが二重加算**、しかも精算後の点で順位を再ソート（順位が変わる） |
| 設定読込 | 履歴画面が**存在しないフィールド**を読み、常に既定値で計算していた |

| グラフ | 「推移」と「累計」が**同じ内容**（どちらも素点の累計） |
| 品質 | **テスト0件**・テスト実行環境なし・デバッグログ散在・デッドコード多数 |
| デザイン | 素の Bootstrap で不統一、青と緑が混在、モバイル最適化なし |

## ✨ 新機能（New）

### 登録不要の点数計算ツール `/calc`
- **アカウント不要**でその場で精算（ウマ・オカ・返し点・焼き鳥）
- 複数半荘の**累計・順位**を自動集計、行削除・全消去
- プレイヤー名編集、**端末内に自動保存**（更新・翌日でも残る）
- 「記録を残したくなったら無料登録」導線つき

### ウマ・オカの自由設定
- ウマ … プリセット（5-10 / 10-20 / なし）＋**順位ごとの自由入力**＋合計0チェック
- オカ … 「自動（返し点−配給原点）／**手動指定**」を切替（`okaOverride`）

### 気軽に使える導線
- ランディングを刷新し「**登録せずに計算する**」を主要導線に
- ゲストでも迷わないヘッダーナビ（計算ツール／ログイン）

### ダークモード
- ライト/ダーク切替をヘッダーに常設、選択は保存、OS設定も尊重（ちらつき防止）

## 🔧 修正（Fixed）

- **精算ロジックを `lib/score.ts` に一本化**（順位・ウマ・オカ・焼き鳥・チップ・累計・検証）
  - オカを明示計算、焼き鳥は分配/総取り両対応＋**ゼロ除算をガード**、ゼロサムを担保
  - ウマ二重加算・順位の再ソートを解消（順位は素点で確定）
- **対局確定（履歴）** を正しい精算で保存（totalScore/rank）
- **対局作成**の不整合（`isCurrentUser`除去・`yakitoriMode`保存・配列送信に統一）
- **グラフを分離**：「半荘ごとの得点」と「累計得点」
- **保存後の同期**（入力後に履歴・グラフが更新されない不具合）を `refreshToken` で解消
- **入力バリデーション**（素点合計＝配給原点×4、ウマ合計0）を追加
- `yakitoriMode` を `'distribution' | 'winner_takes_all'` に統一
- 設定モーダルの誤った制約（焼き鳥点が3で割り切れ必須）を撤廃
- 入力途中の半荘も保存し、更新で消えないよう改善
